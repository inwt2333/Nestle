# 🍼 雀巢 bC 一体化母婴数字化平台 (演示版)

欢迎使用“雀巢 bC 一体化母婴平台”体验项目！本项目旨在通过数字化工具，赋能下沉市场的母婴轻业态实体店（B端），并通过精细化运营与环保活动链接最终消费者（C端）。

本说明专为**非技术人员**编写，只需按照以下简单的三步，您就可以在自己的电脑上完整体验整个系统的核心功能。

---

## ✨ 核心业务体验亮点

本演示系统主要包含以下“杀手级”功能，供您在启动后亲自体验：

1. **🤖 AI 自动化营销 (赋能店员)**
   - **痛点**：下沉市场店员没时间、不会写营销文案。
   - **体验**：系统每天自动扫描顾客档案。以体验版为例，系统发现“张女士的宝宝今天刚满6个月，且有轻度牛奶过敏”，后台会自动调取大语言模型（如 ChatGPT / Gemini），生成符合雀巢超启能恩2段活动、且语气亲切的几十字专属微信话术。店员只需点击“一键发送”即可。
2. **♻️ 环保“罐爱计划” (双向引流)**
   - 顾客带空罐来店，店员在首页点击“扫码回收”，双方即可自动获得积分与返利。
3. **📊 极简门店看板**
   - 大字号、高对比度的“今日销售额”、“回收计数”与“临期预警”，让店长对经营状况一目了然。

## 🚀 如何在电脑上启动这个系统？

请确保您的电脑已经安装了 **Node.js** 和 **Docker Desktop**（Docker需处于全天候开启运行状态）。接下来，请跟着做：

### 第一步：配置 AI API（可选）
1. 在项目文件夹中找到一个名叫 `.env` 的文件，用任何文本编辑器（或记事本）打开它。
2. 找到 `OPENAI_API_KEY=` 这一行。
3. 将等号后面的内容替换为您自己真实的大模型 API 密钥（如果您不填，系统也可以运行，只是会返回一句系统默认的“兜底”话术，而不能生成智能个性化分析）。

### 第二步：一键初始化数据库（仅第一次需要）
1. 打开电脑的“终端” (Mac 用户请按 `Command + 空格` 搜索“终端”或“Terminal”并打开)。
2. 在终端里，输入 `cd `（注意 cd 后面有个空格），然后把整个项目文件夹拖进终端窗口里，按下回车键（这会让你进入项目目录）。
3. 复制并粘贴以下命令，按下回车键：
   ```bash
   npm run db
   ```
   *(后台拉起 Postgres，建表并植入供测试的门店和顾客数据)*

### 第三步：每天日常使用的“一键启动指令”
此后您每次想开启前后端并向别人演示这个看板时，**只需这简简单单的一行指令**：
```bash
npm run dev
```
*(按下回车后，系统会自动同时运行后端的 AI 大脑和前端的可视化看板页面)*

## 📱 恭喜您，可以开始使用了！

一切启动就绪后，请打开您最常用的浏览器（推荐使用 Chrome 谷歌浏览器）：

### 1. 体验门店店员的日常 (核心看板页面)
👉 **在浏览器地址栏输入：** `http://localhost:5173/`

- 这是我们为您制作的高保真模拟小程序界面。
- **推荐操作**：在这个网页上按下键盘的 `F12`，点击图框左上角的“手机/平板”切换图标，将网页设为手机尺寸预览（比如 iPhone 14 Pro 尺寸），这样观感最符合实际店员的手机界面！
- 您会看到针对“张女士”的 AI 专属任务已经陈列在看板上了。

### 2. 体验后台 AI 定时触发大模型
👉 **在浏览器地址栏输入：** `http://localhost:3000/trigger-scrm`

- 实际业务中，AI 排查和写文案是在每天凌晨 2 点。
- 为了演示方便，只要您每次刷新上面这个网址，系统就会模拟执行一次“凌晨扫描”。您可以立即切回到第一个终端窗口里，亲眼看着后台是如何调用 AI 一步步思考并把文案生成出来的！

---
遇到任何黑框报错，请首先检查 Docker 是否处于“Running”状态。祝您体验愉快！

# Step 2: 核心业务流程设计 (API 规划)

## A. O2O 订单路由机制 (LBS 派单策略)

**业务场景**：C端消费者在商城下单时，订单不直接流向总仓，而是基于地理位置推给最近的 B端门店，实现极速达与同城流量转化。

### 时序逻辑描述：
1. **订单生成 (`POST /api/c/orders`)**
   - 顾客提交购物车结算，传入当前地理位置坐标 `(lat, lng)`。
2. **LBS 门店匹配算法**
   - 后端利用 Redis GEO 或 PostGIS 空间查询能力，拉取周边 5-10 公里内的营业门店 (`Store`)。
   - 检测筛选门店的新鲜库存 (`Inventory`) 必须大于订单所需数量。
   - 综合门店距离、历史接单响应时长、门店星级，取最优 `Top 1` 门店。
3. **订单分发与锁定**
   - 将订单状态置为 `PENDING_ACCEPT`（待接单），并将 `storeId` 绑定到该订单。
   - 利用 Redis 预扣减该门店对应的商品库存（5分钟锁定），防止超卖。
4. **B端实时响应**
   - 通过 WebSocket 消息通道向门店店员/店长小程序推送新订单提醒（"您有新的同城订单，请在5分钟内确认"）。
5. **接单与履约 (`POST /api/b/orders/:orderId/accept`)**
   - **接单**：状态更新为 `PREPARING`（备货中），DB库存正式扣减。
   - **拒单/超时**：释放在 Redis 中的预扣减库存，触发降级重路由寻找 `Top 2` 门店。

## B. "一键话术"自动化推送机制 (SCRM 营销)

**业务场景**：利用宝宝档案（出生日期）与大模型，自动替店员写好关怀和营销文案，傻瓜式赋能下沉市场店员。

### 时序逻辑描述：
1. **定时数据扫描 (Cron Job)**
   - 每日凌晨 `02:00`，NestJS 任务调度器运行扫描。
   - 查询 `BabyProfile` 表，计算宝宝的当前月龄。
   - 如：宝宝满 170 天，命中【辅食添加初步】与【奶粉 1 段转 2 段】的营销生命周期规则。
2. **专属员工归属查询**
   - 查询该 `Customer` 最近到店消费的记录，绑定对应的专属店服人员 (`employeeId`)。
3. **AI 智能话术生成任务编排**
   - 整合宝宝信息（如 牛奶过敏史、年龄）和活动信息请求 LLM 接口：
     *Prompt*: "基于系统规则，这是一位对牛奶蛋白过敏的5个半月宝宝妈妈，即将换二段奶粉。请用亲切热情的口吻，生成不超过50字的微信客服沟通话术，并附带本月超启能恩2段买大送小的活动邀请。"
   - 获取大模型返回值，持久化为一条 B端待办任务 (`Task`)。
4. **B端店员端接收与交互**
   - 早上 `09:00`，店员打开小程序 Dashboard。
   - "今日任务" 模块高亮提示："有 3 位顾客需要关怀提醒"。
   - **一键发送流**：点击任务，展示 AI 生成的话术 -> 点击【去发送】 -> 唤起微信 JS-SDK 的 `wx.shareToExternalContact` 或剪贴板复制，带着专属小程序码图片和文本直接分享给C端消费者。
